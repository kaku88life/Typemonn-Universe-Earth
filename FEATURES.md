# Type-Moon Holograph - System Features & Architecture

## 🛠️ 目前功能列表 (Current Features)

### 1. 🌍 3D 全像地球儀 (Holographic Globe)
- **核心引擎**: 基於 `react-globe.gl` (Three.js) 的 3D 渲染。
- **雙層世界**: 支援「表側 (Surface)」與「視界裏側 (Reverse Side)」切換。
- **地形渲染**: 向量式大陸板塊，支援滑鼠懸停高亮。
- **地標系統**: 
  - 不同類型地標顯示不同顏色 (都市=青色, 特異點=紅色, 組織=紫色 等)。
  - 脈衝動畫效果。

### 2. 📍 地點詳情與 FGO 整合 (Location Intelligence)
- **資訊面板**: 顯示地點名稱、座標、描述。
- **FGO 角色連動**: 
  - 自動從 Atlas Academy API 抓取相關從者。
  - **排序**: A-Z 字母排序。
  - **介面**: 緊湊型列表視圖。
- **關聯資訊**: 顯示該地點所屬的作品世界線 (Fate, UBW, FGO 等)。

### 3. ⏳ 時間軸導航 (Temporal Navigation)
- **年代切換**: 底部時間軸可切換不同時代 (神代, 現代, 未來)。
- **歷史過濾**: 地圖上的地點會根據當前選定的年代顯示或隱藏。

### 4. ⚙️ 系統設定 (System Config)
- **國際化 (i18n)**: 支援 英文 (EN) / 繁體中文 (ZH) / 日文 (JP) 即時切換。
- **設定選單**: 齒輪圖示觸發的懸浮選單。

## 📐 介面佈局與限制 (Layout & Restrictions)

你提到的「為什麼沒辦法移動到指定的相對位置」以及目前的限制，主要歸因於本系統採用的 **HUD (Head-Up Display)** 設計哲學。

### 目前的設計邏輯：**固定定位 (Fixed Positioning)**
目前的 UI 面板 (如詳情頁、設定按鈕) 都是使用 CSS 的 `position: fixed` 屬性。

- **優點**: 
  - 像鋼鐵人的頭盔介面一樣，UI 永遠「浮」在 3D 地球上方。
  - 不會因為地球的轉動而跑版。
- **缺點 (目前的限制)**:
  - **缺乏流動性**: 元素之間不知道彼此的存在。例如「左側面板」不會自動避開「底部時間軸」，如果螢幕太小，它們就會重疊。
  - **手動座標**: 我們目前是寫死座標 (例如 `left: 24px`, `top: 50%`)。如果要移動，必須修改程式碼中的座標值。

### 為什麼會感覺「混亂」或「重疊」？
因為瀏覽器視窗大小不同 (例如寬螢幕 vs 筆電螢幕)，固定的 `300px` 面板在小螢幕上可能會佔據太大比例，或者在高度不足時蓋到底部的時間軸。

### 🔧 未來改進建議
若要解決此限制，我們可以引入 **"Responsive Grid Layout" (響應式網格佈局)**：
1. **安全區域 (SafeArea)**: 定義 UI 只能出現在螢幕的邊緣特定區域。
2. **動態高度**: 當螢幕變矮時，自動縮減面板高度。
3. **相對佈局**: 讓面板相對於「其他 UI」排列，而不是相對於螢幕角落。
